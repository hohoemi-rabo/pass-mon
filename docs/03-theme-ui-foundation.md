# 03: テーマ・UI 基盤の構築

## 概要

要件定義のカラーパレットとシニア向け UI/UX 制約に基づき、アプリ全体のデザインシステムを構築する。

## 対応する要件

- REQUIREMENTS.md ⑦ UI/UX デザイン方針
- CLAUDE.md テーマ・カラー / UI/UX の制約

## タスク

- [ ] `constants/theme.ts` をカラーパレットで更新（プライマリ #FF8C42, セカンダリ #4ECDC4, 背景 #FFF8F0 等）
- [ ] NativeWind の `tailwind.config.js` にカスタムカラー・フォントサイズを定義
- [ ] 基本フォントサイズの設定（最小 18px, 見出し 22px 以上）
- [ ] タップ領域の共通スタイル定義（最小 48x48dp）
- [ ] カード型コンポーネントの作成（角丸、やさしい影）
- [ ] ボタンコンポーネントの作成（プライマリ / セカンダリ / 危険）
- [ ] 入力フォームコンポーネントの作成（大きめテキスト、明確なラベル）
- [ ] WCAG AA コントラスト比の検証
- [ ] アプリアイコン・スプラッシュスクリーンのカラー更新

## 完了条件

- カラーパレットがアプリ全体に適用されている
- 全コンポーネントがシニア向けサイズ要件を満たしている
- WCAG AA コントラスト比を満たしている
